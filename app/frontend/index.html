Why does each model need its own packages?

Different model families use different inference libraries. FLUX 1 / FLUX 2 / Klein run through mflux on MLX, Qwen-Image-Edit uses transformers, and future engines may add their own. The catalog mixes models from different ecosystems so you can pick the right tool for each task — but each library has to be importable for that model's engine to run.

Good news: installation is a one-time thing. Click Install Generation in the Pinokio sidebar once, restart the server, and every model in those families works permanently. You only ever re-install after Reset or on a new machine.

The "Install Generation" script bundles all the libraries together for simplicity. If you only ever plan to use one engine, the extra packages still install but they're idle — they don't slow anything down.

What is Steps?

The image starts as random noise. Each step nudges it a little closer to matching your prompt. More steps generally means more refined output — with diminishing returns. Distilled models like FLUX.2 klein and FLUX.1 schnell are specifically trained to converge in just a handful of steps; dev models need many more.

Distilled klein ignores guidance internally (forced to 1.0). Tune for dev models.

What is Guidance?

Classifier-Free Guidance (CFG) scale. Higher values make the image follow your prompt more strictly but reduce creativity and can produce burnt or over-saturated results. Lower values are more dreamy/creative but may ignore parts of your prompt. Most dev models work well at 3.5–5.0. Distilled models (klein/schnell) hardcode guidance to 1.0 internally — this slider is effectively ignored.

What is Seed?

The starting random number for the noise. Same seed + same prompt + same settings = pixel-identical image, every time. Useful for reproducing a result or tweaking a prompt while keeping the composition stable. Set to -1 to randomize per generation. After a generation completes, the actual seed used is shown in the output meta — click 🔄 Reuse params on a recent tile to pull that exact seed back into this field.

Submits N generations back-to-back. Each gets a different seed (sequential from the one above, or fresh random if seed = -1).

What is Batch?

How many images to generate from one click. Each runs sequentially (mflux can't run two generations in parallel on the same GPU). Useful for getting variations of the same prompt without click-click-clicking — set batch to 4, hit Generate, walk away.

What is a LoRA?

LoRA stands for Low-Rank Adaptation. It's a small extra file (usually 30 MB – 300 MB) that patches the base model with a specific style, character, or concept — without needing to retrain the whole model.

You stack a LoRA on top of a base model at generation time. The weight (-2 to 2) controls how strongly it pulls the output toward what it was trained on. Typical range is 0.5–1.5; negative weights subtract the trait. You can stack multiple LoRAs at once and they'll combine.

Where to find them: civitai.com hosts most community LoRAs; search for ones tagged with the model family you're using (FLUX). Download the .safetensors file and drop it into app/lora/ on the server, then click ↻ refresh above.

Note: not every LoRA is compatible with every base model. A LoRA trained on FLUX.1 dev usually won't work on FLUX.2 klein. Check the LoRA's page for compatibility info.

        📚 New here? What does 4B / 4-bit / MLX / gated mean?
        

          
Anatomy of a model name

          
A name like FLUX.2 klein 4B — MLX 4-bit tells you four things:

          
    
            
  • FLUX.2 klein — the model family (klein is a distilled, fast variant of FLUX.2)
    • 
            
  • 4B — 4 billion parameters; the "size" of the model. Bigger usually means more detail and broader knowledge but more memory and slower generation.
    • 
            
  • MLX — Apple's machine learning framework. Models marked MLX are pre-quantized and tuned for Apple Silicon's unified memory; they load fast and use memory efficiently.
    • 
            
  • 4-bit — how precisely each number in the model is stored.
    • 
          


          
What does "X-bit" mean for quality?

          

            The model's billions of numbers can be stored at different precisions. Lower-bit
            versions are smaller files and use less memory, at a small quality cost.
          

          
    
            
  • Full (no bit tag) — the original 16-bit-per-number weights. Highest fidelity, biggest file.
    • 
            
  • 8-bit — about half the size of full. Near-original quality. Sweet spot if you have plenty of memory.
    • 
            
  • 6-bit — smaller still. Subtle quality loss, usually fine for most prompts.
    • 
            
  • 4-bit — about a quarter the size. Visible quality loss on close inspection but the speed and memory savings are huge — recommended starting point on 8–16 GB Macs.
    • 
            
  • 3-bit — smallest. Noticeable quality loss. Use only when memory is tight.
    • 
          


          
Distilled vs full ("klein/schnell" vs "dev")

          

            Some FLUX models are distilled: trained to produce a finished image in just
            1–4 denoising steps. They're fast but ignore the guidance control internally
            (locked to 1.0). Examples: FLUX.2 klein, FLUX.1 schnell.
          

          

            Dev models (FLUX.1 dev, FLUX.2 dev) need 20–30 steps and respond
            to guidance properly. Higher quality, slower, larger.
          


          
What does each chip mean?

          
    
            
  • cached All files for this model are on disk and ready to generate from.
    • 
            
  • partial Some files have downloaded but the model isn't usable yet. Click Download again to resume.
    • 
            
  • not downloaded No files for this model are on disk. Click Download to fetch them.
    • 
            
  • gated The model's repo on Hugging Face requires you to accept a license and provide a personal access token. You'll be prompted for the token in the Download dialog.
    • 
            
  • mlx Pre-quantized for Apple Silicon. Loads fast, runs efficiently on M-series chips. If you see this chip, it's a good pick for Mac.
    • 
          


          
Which one should I pick first?

          

            On any Apple Silicon Mac with ≥8 GB unified memory: start with
            AITRADER/FLUX2-klein-4B-mlx-4bit. ~2.3 GB on disk, fast loads,
            ungated. If quality matters more than speed and you have ≥16 GB: try the MLX 8-bit.

What's the difference between Link and Move?

      
    
        
  • 
          Link creates a symlink under our HF cache pointing at the
          source. Instant, zero copy, but if you later uninstall the source app,
          the link breaks. Best when you want to keep both Pinokio apps installed
          and share the same on-disk cache.
        
    • 
        
  • 
          Move physically relocates the folder into our HF cache.
          On the same disk it's an instant rename (no copy), across disks it falls back
          to copy+delete. Afterward, the source is gone — you can safely delete the
          old launcher entirely. Best for migrating off another launcher.
        
    •

        Why bother setting one?
        

          A Hugging Face token is your personal pass to the HF model hub. Without one,
          you're an anonymous downloader — which means:
        

        
    
          
  • Lower rate limits. HF throttles anonymous traffic far more aggressively. Big downloads stall or hit caps.
    • 
          
  • Slower CDN tier. Authenticated requests get priority routing to closer edge servers.
    • 
          
  • No access to gated models. FLUX.2 dev, FLUX.2 klein base, FLUX.1 dev — all gated. Without a token you literally can't download them, even after accepting their license on the HF site.
    • 
          
  • No access to your own private models. If you've trained or uploaded private LoRAs/checkpoints, the token gives you access here too.
    • 
        

        

          The token is stored locally in app/backend/settings.json.
          It never leaves your Mac except as an Authorization header sent
          directly to huggingface.co during downloads. We never log it.

  1. Go to huggingface.co/settings/tokens (sign up free if you don't have an account)
    2. 
          
  2. Click Create new token
    3. 
          
  3. Token type: Read is enough. Don't pick "Write" or "Fine-grained" unless you know you need them.
    4. 
          
  4. Name it something memorable (e.g. imagestudio-mac)
    5. 
          
  5. Copy the token (starts with hf_…) and paste it below
    6. 
        

        

          For gated repos like black-forest-labs/FLUX.2-dev, you also need
          to visit each repo page once and click "Agree and access" to accept their license.
          That's a one-time step per repo.

        How does direct API access work here?
        

          Our server is configured to bind on 0.0.0.0 at a fixed port
          , which means it listens on every
          network interface (loopback + LAN + Tailscale). Any device that can reach
          this Mac on that port can call the API directly — no Pinokio proxy needed,
          no port-rotation surprises.
        

        

          That's a deliberate trade-off from the "127.0.0.1-only secure default" you
          might see in other apps:
        

        
    
          
  • 
            Pro: stable URL, lower latency, full
            /docs Swagger access from any device, easy scripting.
          
    • 
          
  • 
            Con: the API has no built-in authentication right now,
            so anything on the network you bind to can call it. On a
            personal Tailscale tailnet (only your own devices) that's fine — but
            don't expose port  to the
            public internet without adding an auth layer first.
          
    • 
        

        

          To revert to the "secure default" (loopback only), edit
          start.js and change --host 0.0.0.0 back to
          --host 127.0.0.1. Then access becomes Pinokio-UI-only again.
