            Distilled models (klein, schnell) work great at 4. Dev models want 20–30.
          

          

            What is Steps?
            

              The image starts as random noise. Each step nudges it a little closer
              to matching your prompt. More steps generally means more refined output —
              with diminishing returns. Distilled models like FLUX.2 klein and FLUX.1 schnell
              are specifically trained to converge in just a handful of steps; dev models
              need many more.
            

          

        


        
        

          
          
          

            Distilled klein ignores guidance internally (forced to 1.0). Tune for dev models.
          

          

            What is Guidance?
            

              Classifier-Free Guidance (CFG) scale. Higher values make the image follow
              your prompt more strictly but reduce creativity and can produce burnt or
              over-saturated results. Lower values are more dreamy/creative but may
              ignore parts of your prompt. Most dev models work well at 3.5–5.0.
              Distilled models (klein/schnell) hardcode guidance to 1.0 internally —
              this slider is effectively ignored.
            

          

        


        
        

          
          
          

            What is Seed?
            

              The starting random number for the noise. Same seed + same prompt + same
              settings = pixel-identical image, every time. Useful for reproducing a result
              or tweaking a prompt while keeping the composition stable.
              Set to -1 to randomize per generation. After a generation completes,
              the actual seed used is shown in the output meta — click 🔄 Reuse params on
              a recent tile to pull that exact seed back into this field.
            

          

        


        
        

          
          
          

            Submits N generations back-to-back. Each gets a different seed (sequential
            from the one above, or fresh random if seed = -1).
          

          

            What is Batch?
            

              How many images to generate from one click. Each runs sequentially (mflux
              can't run two generations in parallel on the same GPU). Useful for getting
              variations of the same prompt without click-click-clicking — set batch to 4,
              hit Generate, walk away.

            What is a LoRA?
            

              LoRA stands for Low-Rank Adaptation. It's a small extra file
              (usually 30 MB – 300 MB) that patches the base model with a specific style,
              character, or concept — without needing to retrain the whole model.
            

            

              You stack a LoRA on top of a base model at generation time. The
              weight (-2 to 2) controls how strongly it pulls the output toward
              what it was trained on. Typical range is 0.5–1.5; negative weights subtract
              the trait. You can stack multiple LoRAs at once and they'll combine.
            

            

              Where to find them: civitai.com
              hosts most community LoRAs; search for ones tagged with the model family
              you're using (FLUX). Download the .safetensors file and drop it
              into app/lora/ on the server, then click ↻ refresh above.
            

            

              Note: not every LoRA is compatible with every base model. A LoRA trained
              on FLUX.1 dev usually won't work on FLUX.2 klein. Check the LoRA's page for
              compatibility info.

            MLX, MPS, 4-bit, and gated explained
            
Apple MLX is the native runtime used by mflux. Pre-quantized MLX weights load fastest.

            
PyTorch / MPS runs Diffusers models through Apple's Metal GPU support.

            
4-bit and 8-bit trade a little precision for smaller downloads and lower memory use.

            
Gated models require license acceptance and a Hugging Face token.

Import existing weights

      

        Symlinks an existing HF-cache-style folder into this server's HF cache,
        so you don't re-download models you already have on disk. No copying.
      

    


    

      
What's the difference between Link and Move?

      
    
        
  • 
          Link creates a symlink under our HF cache pointing at the
          source. Instant, zero copy, but if you later uninstall the source app,
          the link breaks. Best when you want to keep both Pinokio apps installed
          and share the same on-disk cache.
        
    • 
        
  • 
          Move physically relocates the folder into our HF cache.
          On the same disk it's an instant rename (no copy), across disks it falls back
          to copy+delete. Afterward, the source is gone — you can safely delete the
          old launcher entirely. Best for migrating off another launcher.
        
    •

        Why bother setting one?
        

          A Hugging Face token is your personal pass to the HF model hub. Without one,
          you're an anonymous downloader — which means:
        

        
    
          
  • Lower rate limits. HF throttles anonymous traffic far more aggressively. Big downloads stall or hit caps.
    • 
          
  • Slower CDN tier. Authenticated requests get priority routing to closer edge servers.
    • 
          
  • No access to gated models. FLUX.2 dev, FLUX.2 klein base, FLUX.1 dev — all gated. Without a token you literally can't download them, even after accepting their license on the HF site.
    • 
          
  • No access to your own private models. If you've trained or uploaded private LoRAs/checkpoints, the token gives you access here too.
    • 
        

        

          The token is stored locally in app/backend/settings.json.
          It never leaves your Mac except as an Authorization header sent
          directly to huggingface.co during downloads. We never log it.
        

      


      

        How to get one (30 seconds)
        
    
          
  1. Go to huggingface.co/settings/tokens (sign up free if you don't have an account)
    2. 
          
  2. Click Create new token
    3. 
          
  3. Token type: Read is enough. Don't pick "Write" or "Fine-grained" unless you know you need them.
    4. 
          
  4. Name it something memorable (e.g. imagestudio-mac)
    5. 
          
  5. Copy the token (starts with hf_…) and paste it below
    6. 
        

        

          For gated repos like black-forest-labs/FLUX.2-dev, you also need
          to visit each repo page once and click "Agree and access" to accept their license.
          That's a one-time step per repo.

Network access — how to reach this app from another device

      

        Where the API is currently reachable, auto-detected. Use these from your main
        Mac, iPhone, scripts, or anywhere else on your LAN / Tailnet.
      


      

        
Bound port

        

          
          
        

        

          Hardcoded in start.js so it survives every restart. If you
          ever want to change it, edit the --port arg there and Stop → Start.
        

      


      

        How does direct API access work here?
        

          Our server is configured to bind on 0.0.0.0 at a fixed port
          , which means it listens on every
          network interface (loopback + LAN + Tailscale). Any device that can reach
          this Mac on that port can call the API directly — no Pinokio proxy needed,
          no port-rotation surprises.
        

        

          That's a deliberate trade-off from the "127.0.0.1-only secure default" you
          might see in other apps:
        

        
    
          
  • 
            Pro: stable URL, lower latency, full
            /docs Swagger access from any device, easy scripting.
          
    • 
          
  • 
            Con: the API has no built-in authentication right now,
            so anything on the network you bind to can call it. On a
            personal Tailscale tailnet (only your own devices) that's fine — but
            don't expose port  to the
            public internet without adding an auth layer first.
          
    • 
        

        

          To revert to the "secure default" (loopback only), edit
          start.js and change --host 0.0.0.0 back to
          --host 127.0.0.1. Then access becomes Pinokio-UI-only again.

        Are these addresses stable?
        
    
          
  • 
            App port ():
            Yes — pinned in start.js.
